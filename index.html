Why Testing

Catch bugs early on.

Makes code safer to change.

Automation.

Consistent.

Act as documentation.

Forces you to write better code.

Types of Testing

  • Unit Testing
    • Performed by developer.
    • Grab small components in the app and testing them in isolation of other components.

Types of Testing

  • End to End Testing
    • Testing the entire flow of application from the users point of view.

Types of Testing

  • Manual Testing
    • Testing the software for correct behavior prior to release to end users.

Types of Testing

  • Stress Testing
    • Testing activity that determines the robustness of software by testing beyond the limits of normal operation.

Test Driven Development (TDD)

Write your tests before coding and start making that test pass

"Fail Test - Pass Test - Refactor"

Autotest all your tests each time you update your code

Existing tests keep track of previously-fixed bugs

Test as many browsers and devices as you have access to

Wait for your tests or CI server to yell at you

What is (fake) TDD?

Keep fixing your code until you can't see any bugs

Test using alerts and console logs

Hotswap variables directly to corner bugs

Refresh the page as much as you can

If I can't see the bug then it's not there

Only bother testing with Chrome

Wait for your manager to yell at you

Install Jasmine, Karma and Bower as dev dependency npm install karma jasmine --save-dev

Install front end essentials (angularjs, angular-mocks) using bower bower install angular angular-mocks --save
